# 🛡️ תיעוד מערכת ההרשאות - פרויקט דוח-1

## 📋 סיכום המצב הנוכחי (מרץ 2026)
שחזרנו את המערכת למצב עבודה מלא לאחר תיקון באגים קריטיים במנגנון האימות ובמבנה הדפים.

### 🔑 מפתחות הרשאות (Roles)
המערכת מפרידה כעת בצורה מוחלטת בין שני סוגי הרשאות:
1.  **`phone`**: הרשאה לצפייה באנשי קשר וטלפונים (מוגנת ע"י `AuthGuard.tsx`).
2.  **`update`**: הרשאה לעדכון נוכחות (מוגנת ב-`update.html`).

> [!IMPORTANT]
> הבקשה לשרת נשלחת כעת עם השדה **`role`** (עם 'l' אחת) כדי להתאים לציפיות ה-Backend עבור קלט (POST). השרת עשוי להחזיר בתגובה שדה בשם `roll`, אך הקוד יודע לטפל בשניהם.

---

## 🛠️ תיקונים קריטיים שבוצעו
1.  **תיקון מבנה ה-CSS:** תגית ה-`<style>` ב-`update.html` נסגרה בטעות באמצע הקוד. הדבר גרם לכך שכל העיצובים (כולל הסתרת הודעות שגיאה) לא עבדו. **התגית תוקנה ונסגרה בסוף הבלוק.**
2.  **שיפור הודעות שגיאה:** הוספנו מספרי סטטוס (כמו 401) להודעות השגיאה ב-UI כדי להבחין בין בעיית הרשאות לבעיית תקשורת.
3.  **תיקון תאימות גוגל (CORS/Origin):** הוגדר פורט 8080 לבדיקות מקומיות. יש לוודא שפורט זה מורשה ב-Google Cloud Console.

## 🧪 בדיקות אוטומטיות (Vitest)
הוספנו סט בדיקות כדי לוודא שהבאגים האלו לא יחזרו:
- `src/test/AuthGuard.test.tsx`: בודק שהשדה `role` נשלח נכון ושמנגנון האימות מגיב נכון לסטטוסים מהשרת.
- `src/test/attendanceUtils.test.ts`: בודק את לוגיקת עיבוד הנתונים.

### הרצת בדיקות (לפני כל פריסה!):
```bash
npm run test
```

---

## 🚀 תהליך עדכון ופריסה (Deploy)
כדי לעדכן את האתר החי ב-GitHub Pages, יש להריץ את הרצף הבא:
1. שמור את כל הקבצים.
2. הרץ בטרמינל: `npm run deploy`.
   *(הפקודה בונה את הפרויקט אוטומטית ומעלה ל-gh-pages).*
3. וודא שב-GitHub ה-Main Branch מעודכן (`git push`).

---

> [!TIP]
> אם המשתמש מדווח על "אין הרשאה" למרות שהוא מחובר, בדוק קודם כל את ה-Role שלו ב-n8n או במסד הנתונים. המערכת כרגע קפדנית מאוד בנושא זה.
